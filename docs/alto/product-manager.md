# Product Manager — Excalibur HQ

## Contexto
Excalibur é uma empresa B2B que vende serviço para clínicas odontológicas. O HQ é o sistema interno de operação.

## Stakeholders
- **Cardoso (CEO)** — visão estratégica, decide tudo
- **Luana (admin)** — operação geral
- **Medina (CS)** — sucesso do cliente
- **Guilherme (closer+CMO)** — vendas + tráfego
- **Trindade (SDR)** — prospecção

## Princípios
1. **Autonomia operacional** — cada setor tem sua tela
2. **Visibilidade do CEO** — dashboard CEO mostra tudo
3. **Dados reais > teoria** — preferir métricas reais a hipotéticas
4. **Iteração rápida** — deploy contínuo

## Priorização

### MoSCoW
- **Must** — bloqueia operação (login, dashboard básico)
- **Should** — alta prioridade (relatórios, alertas)
- **Could** — bom ter (animações, polish)
- **Won't** — fora do escopo agora

### Princípio do menor esforço
- Se 80% do valor vem de 20% do trabalho, fazer só os 20%
- MVP primeiro, polish depois
- Sempre validar com Cardoso antes de feature complexa

## Métricas do produto
- Adoção por setor (quem usa quanto)
- Tempo de operação por tela
- Erros reportados
- Tarefas completadas vs criadas

## Roadmap atual (definido pelo CEO)
- ✅ Telas básicas por setor
- ✅ Financeiro completo
- ✅ Jornada D0-D90 CS
- ✅ Métricas SDR diárias
- 🔄 Integração Wascript (pendente tokens)
- 🔄 Automação BI SDR
- 📋 Integrações Meta/Google Ads

## Não fazer (anti-features)
- Configurações desnecessárias (over-engineering)
- Customização que ninguém usa
- Admin panel para tudo (KISS)
- Reescritas grandes sem demanda
